\# Файловый менеджер на Python



\## Описание

Консольный файловый менеджер с поддержкой нескольких пользователей. Работает в изолированной директории, есть защита от выхода за её пределы.



\## Функции

1\) навигация по папкам (cd, pwd, ls)

2\) создание/удаление папок (mkdir, rmdir)

3\) работа с файлами (touch, read, write, rm, cp, mv)

4\) архивация zip/tar (archive, extract)

5\) псевдографический интерфейс (ls gui)

6\) регистрация и вход пользователей

7\) квоты на дисковое пространство (по умолчанию 100 мб)



\## Установка

```bash

git clone https://github.com/Ivanpython2006/file-manager.git

cd file-manager

python file\_manager.py

```



\## Настройка

В файле config.json можно изменить рабочую директорию:

```json

{

&#x20;   "working\_directory": "./konder"

}

```



\## Команды

```

Основные команды:

&#x20; pwd                    (показать текущую директорию)

&#x20; ls                     (показать содержимое обычный режим)

&#x20; ls gui                 (показать содержимое псевдографика)

&#x20; cd <путь>              (перейти в директорию)

&#x20; cd /                   (перейти в корень)



Работа с файлами:

&#x20; touch <имя>            (создать пустой файл)

&#x20; read <имя>             (прочитать файл)

&#x20; write <имя> <текст>    (записать текст в файл)

&#x20; rm <имя>               (удалить файл)

&#x20; cp <откуда> <куда>     (скопировать файл)

&#x20; mv <откуда> <куда>     (переместить или переименовать)



Работа с папками:

&#x20; mkdir <имя>            (создать папку)

&#x20; rmdir <имя>            (удалить папку)



Архивация:

&#x20; archive <имя> <архив>  (заархивировать zip или tar)

&#x20; extract <архив> \[путь] (распаковать)



Пользователи:

&#x20; register <имя> <пароль> (регистрация)

&#x20; login <имя> <пароль>    (вход)

&#x20; logout                  (выход)

&#x20; quota \[имя] \[нов\_квота] (просмотр или изменение квоты)



&#x20; help                    (показать эту справку)

&#x20; exit                    (выход)

```



\## Как это работает

1\) при регистрации создается личная папка пользователя

2\) все операции проверяются на выход за пределы рабочей директории

3\) перед записью проверяется квота

4\) пользователи и пароли хранятся в users.json



\## Требования

библиотеки из стандартной поставки (os, shutil, json и т.д.)
